# Mebius Code 项目管理文档

## 1. 成员与分工

本项目为个人独立项目，开发者承担需求分析、系统设计、后端开发、前端开发、测试、部署和文档编写工作。

| 成员 | 角色 | 主要工作 | 贡献占比 |
| --- | --- | --- | --- |
| 个人开发者 | 项目负责人 / 全栈开发者 | 选题、需求分析、架构设计、NestJS 后端、Vue 前端、Bun/OpenTUI TUI、Kotlin/Jetpack Compose Android 伴随客户端、数据库实体、Agent 工作流、安全策略、测试、部署和发布维护 | 100% |

## 2. AI 应用情况

项目开发过程中使用 AI 工具辅助完成工程工作，但由开发者决定需求边界、技术取舍、代码取舍和最终提交内容。

AI 主要用于：

- 分析项目约束，确认系统需要包含 Server 端、Client 端和智能化能力。
- 收敛选题范围，将“智能编程 Agent 平台”拆分为模型配置、项目工作区、会话、工具调用、审批和审计等模块。
- 辅助设计后端模块边界、实体关系和关键接口。
- 辅助生成 NestJS 服务、控制器、DTO、实体和单元测试初稿，再由开发者结合项目实际修改。
- 辅助规划 Vue 工作区界面、状态管理和 SSE 事件处理。
- 辅助对比 OpenCode、Claude Code 和 Codex 的 CLI/TUI 工作流，确定 TUI 作为第二客户端复用后端能力的方案。
- 辅助梳理安全策略，包括 API Key 加密、路径沙箱、命令策略、会话命令授权、工具审批和审计日志。
- 辅助设计技能系统，包括技能发现优先级、选择逻辑、Slash 命令自动补全和活动技能注入模型上下文。
- 辅助设计外部上下文能力，包括默认 Web 检索工具、远程 MCP Server 接入、Context7 预设和 TUI MCP 浏览界面。
- 辅助梳理仓库级 `AGENTS.md` 项目指令、回合级撤销/重做、Plan 讨论/修订和工具审批后恢复执行流程。
- 辅助设计 Android 伴随客户端、移动端概览 API 和移动端会话体验，复用后端能力到轻量移动入口。
- 辅助完善多端消息渲染体验，包括 Web/Android 数学公式渲染、TUI 终端公式转换、工具消息摘要和回合完成后的 transcript 同步。
- 辅助完善账号体验，包括注册资料昵称化、Web 账户安全页、密码修改接口、前端表单校验和后端密码安全测试。

AI 没有直接决定项目总结内容。项目总结部分按课程要求保留给开发者自行补充。
