# Mebius Code 项目管理文档

## 1. 成员与分工

本项目为个人独立项目，开发者承担需求分析、系统设计、后端开发、前端开发、测试、部署和文档编写工作。

| 成员 | 角色 | 主要工作 | 贡献占比 |
| --- | --- | --- | --- |
| 个人开发者 | 项目负责人 / 全栈开发者 | 选题、需求分析、架构设计、NestJS 后端、Vue 前端、数据库实体、Agent 工作流、安全策略、测试、部署和文档维护 | 100% |

## 2. AI 应用情况

项目开发过程中使用 AI 工具辅助完成工程工作，但由开发者决定需求边界、技术取舍、代码取舍和最终提交内容。

AI 主要用于：

- 分析课程设计要求，确认项目需要包含 Server 端、Client 端和智能化能力。
- 收敛选题范围，将“智能编程 Agent 平台”拆分为模型配置、项目工作区、会话、工具调用、审批和审计等模块。
- 辅助设计后端模块边界、实体关系和关键接口。
- 辅助生成 NestJS 服务、控制器、DTO、实体和单元测试初稿，再由开发者结合项目实际修改。
- 辅助规划 Vue 工作区界面、状态管理和 SSE 事件处理。
- 辅助梳理安全策略，包括 API Key 加密、路径沙箱、命令策略、工具审批和审计日志。
- 辅助维护系统需求文档、系统设计文档和项目管理文档。

AI 没有直接决定项目总结内容。项目总结部分按课程要求保留给开发者自行补充。

## 5. 测试与质量保障

项目通过后端单元测试覆盖关键业务逻辑：

- 用户偏好合并与默认值归一化。
- 模型 Provider 搜索、连接校验、API Key 加密更新和错误处理。
- OpenAI-compatible 流式与非流式响应解析。
- Agent 工具循环、待审批阻塞、审批后恢复和工具消息还原。
- 项目删除、文件增删改重命名、ZIP 导入、Git 状态、暂存、提交和推送。
- 命令策略、项目级命令授权和危险命令语法拒绝。
- 路径沙箱、敏感目录屏蔽和精确项目根删除。
- 审计日志按角色范围查询。

前端通过 TypeScript 类型约束、Vite 构建和 `vue-tsc` 类型检查保障基础质量。
