# Mebius Code 系统需求文档

## 1. 项目背景与可行性

Mebius Code 是面向开发者的多端智能编程 Agent 平台。系统通过大语言模型、代码工作区、工具调用和用户审批机制，帮助开发者完成需求分析、计划制定、代码阅读、文件修改、命令执行和上下文管理。

`Mebius` 的命名来源承载了友情、羁绊与共同成长的主题；在本系统中，它被转化为开发者与智能 Agent 共同完成复杂编程任务的协作关系。

## 2. 领域边界

系统聚焦于“智能编程辅助”领域，不覆盖通用办公聊天、完整 DevOps 平台或商业级 IDE 的所有能力。核心边界包括：

- 用户、项目、模型配置、会话和 Agent 任务管理。
- 服务端代码工作区中的文件读取、搜索、补丁生成和命令执行。
- Plan Mode、工具审批、会话清理和上下文压缩。
- Web、CLI 和后续桌面端共享同一后端数据。

## 3. 目标用户

- 个人开发者：希望通过 Agent 加速阅读、修改和测试代码。
- 学生开发者：希望在课程项目中使用 AI 完成工程化开发任务。
- 项目维护者：希望记录 AI 协作过程、审查修改计划并保留审计记录。

## 4. 竞品分析

- Claude Code：CLI 体验成熟，适合本地项目协作，但主要围绕 Claude 模型生态。
- OpenCode：支持多模型 Provider 和服务端模式，开放性强。
- Codex：强调 Agentic Coding 工作流、Plan Mode 和工具调用。

Mebius Code 的课程项目优势是范围清晰、可解释性强、文档可完整呈现面向对象建模；劣势是功能深度和商业产品稳定性无法与成熟工具相比。

## 6. 客户端与服务端职责

服务端负责认证鉴权、数据持久化、模型调用、Agent 编排、工具执行、审批和审计。客户端负责项目选择、会话交互、Plan 展示、工具审批、文件 Diff 展示和运行日志展示。
